## What Phase 7 Does

Phase 7 takes all small-molecule (SM) candidates generated by Phase 5 and all biologic candidates from Phase 6 and attempts to improve them simultaneously across multiple competing objectives through an active-learning Bayesian optimization loop. The key insight is that drug discovery is inherently multi-objective: a compound that is extremely potent but toxic, or potent and safe but identical to a known drug, is not useful. Phase 7 uses a Gaussian Process (GP) surrogate to model the objective landscape and a Pareto front to track which candidates are non-dominated — the set of candidates where no other candidate is strictly better on every objective at once.

The loop runs until the hypervolume of the Pareto front stops improving meaningfully (convergence criterion: < 1% improvement per iteration), or until the maximum iteration budget is exhausted.

**Trigger condition:** runs if `phase5_output is not None OR phase6_output is not None`.

## Objective Space

### Small Molecules (SM)

| Objective Key | Description | Formula | Range |
|---|---|---|---|
| `potency` | Vina binding affinity | `vina_norm` from P5 (clamped to [0,1]) | [0, 1] |
| `admet_score` | ADMET safety/absorption/metabolism | `score_admet()` from Phase 5 | [0, 1] |
| `novelty` | Chemical novelty vs approved drugs | `1 - max_tanimoto_to_approved` | [0, 1] |

All three objectives are **maximised** and are already normalised to [0, 1] before being passed to the GP.

### Biologics (peptides from Phase 6)

| Objective Key | Description | Source | Range |
|---|---|---|---|
| `developability_score` | Physicochemical developability | `score_developability()` from Phase 6 | [0, 1] |
| `novelty_bio` | Biological novelty | Hard-coded constant 1.0 (biologics are inherently novel vs small drugs) | 1.0 |

The biologic objectives are deliberately simpler — only two — because Phase 6 does not produce a Vina score, and developability already aggregates charge, hydrophobicity, instability index, solubility, and half-life components.

## Active-Learning Loop Architecture

```
for each target symbol in all_targets:
    evaluated = [all P5/P6 candidates for this target]  # initial population
    pool = []

    for iteration in range(P7_MAX_ITER):  # max 5
        # Fit one GP per objective on evaluated set
        fitted = surrogate.fit(evaluated, is_peptide=...)

        # Compute Pareto front and hypervolume
        front, dominated = compute_pareto_front(evaluated, obj_keys)
        hv = compute_hypervolume(front, obj_keys)

        # Convergence check
        if hv_improvement < P7_HV_THRESHOLD (1%):
            break

        # LLM iteration review (gate 7.2_iteration_review) — after iter 0
        _gate_iteration_review(symbol, iteration, evaluated, hv, obj_keys)

        # Generate pool or suggest from existing pool via GP-UCB
        if pool is non-empty:
            suggestions = surrogate.suggest(pool, n_suggest=20)
        else:
            pool = _generate_pool(front, is_peptide)  # BRICS / single-residue mutations
            suggestions = pool[:20]

        # Re-evaluate suggestions (re-ADMET for SM; re-developability for biologic)
        new_evals = _re_evaluate(suggestions, ...)
        evaluated.extend(new_evals)

    # Final Pareto front, desirability scoring, ranking
    final_front = compute_pareto_front(evaluated, obj_keys)[0]
    for c in final_front:
        c["desirability"] = compute_desirability(c, obj_keys)
    final_front.sort(key=desirability, reverse=True)
    return top-20 by desirability
```

### Environment Variables

| Variable | Default | Description |
|---|---|---|
| `P7_MAX_ITER` | 5 | Maximum active-learning iterations per target |
| `P7_N_SUGGEST` | 20 | Candidates suggested per iteration |
| `P7_HV_THRESHOLD` | 0.01 | Hypervolume improvement convergence threshold (1%) |

## GP Surrogate — `gp_surrogate.py`

### Kernel and Fitting

Each objective has an independent `ObjectiveGP` instance backed by `sklearn.gaussian_process.GaussianProcessRegressor`:

```python
kernel = ConstantKernel(1.0) * RBF(length_scale=1.0) + WhiteKernel(0.01)
gp = GaussianProcessRegressor(
    kernel=kernel,
    n_restarts_optimizer=3,
    normalize_y=False,
)
```

- **Scaling:** Both `X` (features) and `y` (objective values) are independently scaled with `StandardScaler` before fitting. This is critical for the RBF kernel's length scale hyperparameter optimization to converge properly.
- **Restarts:** 3 optimizer restarts (`n_restarts_optimizer=3`) to reduce likelihood of landing in local optima of the log-marginal-likelihood surface.
- **Fit threshold:** If fewer than 3 evaluated candidates are available, fitting is skipped and the fallback path is used.

### Feature Representations

**Small molecules (SMILES):**

Primary: Morgan fingerprint, radius 2, 2048-bit, using the `GetMorganGenerator` API (RDKit). The new `rdFingerprintGenerator` API is used rather than the deprecated `AllChem.GetMorganFingerprintAsBitVect` to avoid RDKit deprecation warnings.

```python
gen = GetMorganGenerator(radius=2, fpSize=2048)
fp = gen.GetFingerprintAsNumPy(mol).astype(np.float32)  # shape (2048,)
```

Fallback (if Morgan FP fails — e.g. invalid SMILES): physicochemical descriptor vector of 9 properties (MW, LogP, TPSA, HBD, HBA, RotBonds, Rings, AromaticRings, QED), zero-padded to 128 dimensions.

**Biologics (peptide sequences):**

One-hot encoding over the 20 canonical amino acids (`ACDEFGHIKLMNPQRSTVWY`), padded to `max_len=60` residues. Produces a vector of shape `(60 × 20) = 1200` dimensions. Positions beyond the sequence length are zero-filled.

```python
AA = "ACDEFGHIKLMNPQRSTVWY"   # 20 canonical, alphabetical
vec = np.zeros(60 * 20, dtype=np.float32)
for i, aa in enumerate(sequence[:60]):
    j = aa_idx.get(aa, -1)
    if j >= 0:
        vec[i * 20 + j] = 1.0
```

### Acquisition Function — UCB

Upper Confidence Bound (UCB) with fixed β = 2.0:

```
UCB(x) = μ(x) + β × σ(x)
```

The aggregate acquisition score across all objectives is the **equal-weight sum** of per-objective UCB values, normalised by the number of objectives:

```python
aggregate = sum(gp_k.ucb(X, beta=2.0) for k in objectives) / len(objectives)
```

Candidates in the pool are ranked by this aggregate UCB and the top `P7_N_SUGGEST` (20) are returned as suggestions.

## Pool Generation

### BRICS-Based Analog Enumeration (SM)

After each iteration, if the pool is exhausted, a new pool is generated by calling `generate_with_brics(seed_smiles, n_generate=100)` on the SMILES strings of Pareto-front members. BRICS (Breaking of Retrosynthetically Interesting Chemical Substructures, Degen et al. 2008) fragments the Pareto-front SMILES at BRICS-defined bond types and then randomly recombines fragments from the resulting fragment library.

All generated SMILES are returned as candidate dicts inheriting the parent's metadata, with `parent` set to the seed SMILES.

### Single-Residue Substitution (Biologic)

For each Pareto-front peptide sequence, a number of single-residue substitutions are generated:
- Position: uniformly sampled from [0, len(seq)-1]
- Replacement amino acid: uniformly sampled from all 20 canonical AA
- Number of mutations per parent: `n_mutations // len(front)`

For a typical run with 5 Pareto-front members and 100 pool size, this produces 20 single mutants per parent peptide. The mutation is recorded as `_mutation: {wt_aa}{position+1}{mut_aa}` in the candidate dict for traceability.

## Pareto Front — `pareto.py`

### Dominance Definition

Candidate A **Pareto-dominates** candidate B (under maximisation) if and only if:
1. A is at least as good as B on **every** objective: `all(A_k >= B_k for k in objectives)`
2. A is strictly better than B on **at least one** objective: `any(A_k > B_k for k in objectives)`

The `_dominates()` function implements this in pure Python. `compute_pareto_front()` runs a naive O(n²) scan — adequate for n < 200 candidates per target.

### Hypervolume Indicator

The hypervolume indicator (Zitzler & Thiele 1999) measures the volume of objective space dominated by the Pareto front, bounded below by a reference point. With all objectives normalised to [0, 1], the reference point is the origin `[0, 0, ...]`.

- **2 objectives:** exact WFG sweep-line algorithm, O(n log n)
- **3 objectives (SM):** approximated as the mean of pairwise 2D hypervolumes across the 3 objective pairs — a conservative underestimate but monotonically tracks true HV improvement

```python
hv = mean(hv_2d(points_ij, ref_ij) for i,j in combinations(objectives, 2))
```

The convergence criterion compares consecutive hypervolumes:
```python
hv_improvement = (hv_new - hv_prev) / max(1e-9, hv_prev)
converged = (hv_improvement < P7_HV_THRESHOLD) and (iterations_run > 0)
```

### Desirability Scalar

Within the Pareto front, candidates are ranked by a weighted-sum **desirability** score to produce a scalar ranking for the top-20 output:

| Objective | Weight |
|---|---|
| `potency` | 0.30 |
| `admet_score` | 0.25 |
| `novelty` | 0.15 |
| `developability_score` | 0.15 |
| `selectivity` | 0.15 |

Unrecognised objective keys fall back to equal weight `1/len(objectives)`.

## Re-Evaluation

Re-evaluation runs on suggestions before they are added to the evaluated set:

**SM candidates:**
- Calls `score_admet(smiles, indication_type)` from `src/phases/phase5/admet.py`
- Updates `admet_score` field
- Preserves `vina_norm` from the parent/seed (potency is **not** re-docked in P7 — see Bottleneck H4)

**Biologic candidates:**
- Calls `score_developability(sequence, target_class, indication_type)` from `src/phases/phase6/developability.py`
- Updates `developability_score`, `novelty_bio` (remains 1.0)
- Recomputes `desirability`

## LLM Decision Gate

| Gate ID | Trigger | Prompt |
|---|---|---|
| `7.2_iteration_review_{symbol}_iter{n}` | Every iteration after iter 0, per target | Bayesian optimization review: chemical/sequence space coverage, explore vs exploit balance, flag unreasonable suggestions |

**Schema expected from LLM:**
```json
{
  "space_explored": "...",
  "balance_assessment": "...",
  "flagged_unreasonable": [],
  "recommendation": "..."
}
```

The gate runs at `temperature=0.2`, `max_tokens=300`. It is non-blocking — if the LLM call fails, the loop continues. The gate result is logged to the `decisions` table with `gate = "7.2_iteration_review_{symbol}_iter{iteration}"`.

## Fallback Path

If the GP fails to fit (fewer than 3 evaluated samples, or an exception during fitting), Phase 7 falls back to simple sorting:

```python
def _fallback_optimize(symbol, sm, bio):
    all_cands = sorted(sm + bio, key=lambda c: c.get("desirability", 0.0), reverse=True)
    return {
        "pareto_front": all_cands[:20],
        "iterations_run": 0,
        "hypervolume_final": 0.0,
        "note": "fallback_no_optimization",
    }
```

This ensures Phase 7 always produces a ranked list even when the active-learning loop cannot run.

After each target's optimization, the `candidates` table in Supabase is updated:

```sql
UPDATE candidates
SET subscores = subscores || {"pareto_rank": ..., "desirability": ..., "iterations_run": ...}
WHERE run_id = ? AND target_id = ?
```

The update uses a merge pattern (`{**existing_subscores, **new_fields}`) to preserve all existing subscores from P5/P6 while adding P7-specific fields.

## Performance Notes

| Configuration | Approx. wall time per target | Notes |
|---|---|---|
| 5 targets × 5 iterations × 20 suggestions | 5–30 min (CPU) | Dominated by ADMET re-scoring (sklearn GP itself is fast) |
| Biologic-only (no SM docking) | 2–10 min per target | Developability scoring is cheaper than ADMET |
| Fallback (GP fit fails) | < 5 s | Just a sort operation |

The GP fitting itself (RBF + WhiteKernel, n=20–100 samples, d=2048 features) takes ~0.5–3 s per objective on a modern CPU. With 3 SM objectives, expect 1.5–10 s per `fit()` call. The bottleneck is `score_admet()` (ADMET re-scoring), which is a series of ML model inferences.

## Known Limitations and Bottleneck Cross-References

See `bottlenecks/phase7_phase8.md` for detailed analysis. Key issues:

- **H1 (red):** No BoTorch — sklearn UCB ignores objective correlations and cannot optimize acquisition via gradients. ~15–25% efficiency loss vs qNEHVI.
- **H2 (yellow):** BRICS pool is closed over the Pareto front's scaffold, susceptible to local optima.
- **H3 (yellow):** Single-residue mutations only — no indel or scaffold-swap for biologics.
- **H4 (yellow):** Potency not re-measured — GP extrapolates on stale `vina_norm` values.
- **H5 (green):** One-hot peptide features are high-dimensional for small datasets.
